# 校园快递管理系统

这是一个面向软件工程课程实践的校园快递管理系统文档仓库，重点服务于本地演示、课程答辩和后续实现落地。

## 仓库内容

- `docs/01_项目概述与功能范围.md`：项目定位、角色设计、核心功能和范围边界。
- `docs/02_最小数据库设计.md`：支撑课程演示的最小表结构和字段建议。
- `docs/03_本地演示脚本.md`：答辩或课堂展示时可直接照着走的演示顺序。
- `docs/04_开发落地建议.md`：前后端目录、技术路线和开发阶段建议。
- `docs/05_前后端接口清单与字段约定.md`：用于 AI 协作开发的接口、字段、状态和页面对照约束。
- `docs/README.md`：文档导航和使用建议。

## 精简说明

本仓库已对原始 AI 生成文档做了收敛处理，只保留和课程项目最相关、最容易落地的核心内容：

- 去掉重复表达较多的扩展说明
- 去掉通用教程型材料
- 合并内容高度重叠的设计与演示说明
- 保留最适合课程展示和后续编码的主线文档

## 仓库原则

- `docs/` 是当前唯一维护的主文档目录
- 不再保留与 `docs/` 内容重复的原始长文档副本
- 文档以“够用、清楚、能指导实现”为优先，不追求堆积材料数量

## 项目核心范围

- 登录与角色识别
- 普通用户、快递员、管理员三类角色主页
- 收件入库与用户取件
- 发件申请、审核分配与揽件确认
- 异常件登记与处理
- 基础记录查看

## 当前实现进度

### 后端服务 (`backend/`)

后端服务已基于 **Java 21** 和 **Spring Boot 3.2.5** 搭建完毕，集成了 **MyBatis-Plus**，并完全按照 `docs/05_前后端接口清单与字段约定.md` 完成了主线核心模块的开发：

- **数据库搭建**：已在 `backend/schema.sql` 准备好包含 4 张核心表（`user`、`receive_record`、`send_request`、`exception_record`）与默认账号的建表脚本，本地连接密码默认为 `123`（配置见 `application.yml`）。
- **功能模块**：已跑通用户鉴权模块、收件录入与出库、发件申请与分配揽件、异常登记处理及首页大盘数据统计所有接口。
- **环境兼容性**：代码已去除 Lombok 依赖并替换为原生 Getter/Setter，兼容如 JDK 26 等较新版本 Java 环境下的稳定编译，确保“开箱即用”。
- **启动与测试**：可通过本地 IDE 直接运行 `CampusExpressApplication.java`，或进入 `backend` 目录使用自带的 Maven Wrapper 启动：
  ```bash
  ./mvnw spring-boot:run
  ```
