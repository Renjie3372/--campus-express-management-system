# 项目概述与功能范围

## 1. 项目定位

校园快递管理系统是一个面向软件工程课程实践的轻量级业务系统，用于演示校园快递站点在日常收件、取件、寄件和异常处理场景下的基本管理能力。

本项目的目标不是做成商业化平台，而是完成一套：

- 本地可运行
- 角色清晰
- 流程完整
- 便于答辩展示

的课程项目成果。

## 2. 项目目标

本项目建议围绕以下目标展开：

1. 实现登录与角色识别。
2. 实现快递员收件入库、用户查件和系统自助取件核验的闭环。
3. 实现发件申请、管理员分配、快递员揽件的闭环。
4. 实现异常件登记与处理。
5. 支持基础记录查看，满足课程展示要求。

## 3. 角色设计

建议仅保留 3 类核心角色。

### 3.1 普通用户

对应对象：

- 学生
- 老师

主要功能：

- 登录系统
- 查看自己的快递
- 查看取件码和存放位置
- 提交发件申请
- 查看自己的历史记录

### 3.2 快递员

主要功能：

- 登录系统
- 收件入库
- 录入快递单号、收件人账号、快递公司和存放位置
- 查看收件记录
- 查看被分配的发件任务
- 确认揽件
- 查看已处理记录

### 3.3 管理员

主要功能：

- 登录系统
- 审核发件申请
- 分配快递员
- 登记和处理异常件
- 查看全站业务记录
- 查看用户信息和系统概览

## 4. 核心功能范围

### 4.1 必做功能

- 登录
- 角色识别
- 三类角色不同主页
- 收件入库
- 快递查询
- 自助取件核验
- 发件申请
- 发件审核与分配
- 快递员揽件确认
- 异常登记
- 异常处理
- 基础记录查看

### 4.2 不建议在本阶段纳入的功能

以下内容不影响课程项目核心展示，建议暂不实现或仅在答辩时口头说明：

- 短信或微信提醒
- 扫码枪识别
- 小程序接入
- 批量导入导出
- 报表系统
- 复杂统计分析
- 多级审批
- 复杂权限树

## 5. 三条核心业务主线

### 5.1 收件流程

1. 快递员录入快递信息。
2. 系统生成取件码并保存收件记录。
3. 普通用户查看自己的快递。
4. 普通用户输入快递单号和取件码。
5. 系统自动核验并完成出库。
6. 系统生成取件记录。

### 5.2 发件流程

1. 普通用户提交发件申请。
2. 管理员查看申请并分配快递员。
3. 快递员确认揽件。
4. 系统更新发件状态。

### 5.3 异常处理流程

1. 管理员新增异常记录。
2. 管理员填写处理结果。
3. 系统更新异常状态。

## 6. 建议状态设计

### 6.1 收件状态

- 待取件
- 已出库
- 异常

### 6.2 发件状态

- 已提交
- 已分配
- 已揽件
- 已完成
- 异常

### 6.3 异常状态

- 待处理
- 已处理

## 8. 验收重点

如果以课程实践演示为目标，建议至少确保以下内容稳定可展示：

- 正确密码可登录，错误密码不可登录
- 三类角色进入不同主页
- 收件流程完整
- 发件流程完整
- 异常处理流程完整
- 每类角色都有辅助页面可展示

## 9. 一句话总结

**本项目的最佳范围不是做大做全，而是围绕登录、角色主页、收件、发件、异常和记录查看这几条主线，把系统做成一套可运行、可演示、可答辩的课程实践成果。**
