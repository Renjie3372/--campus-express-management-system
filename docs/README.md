# 文档导航

本目录为“校园快递管理系统”精简版文档集合，面向课程实践场景，强调可展示、可实现、可继续扩展。

## 文档列表

### 1. 项目概述与功能范围

文件：`01_项目概述与功能范围.md`

适合用途：

- 统一项目名称和项目定位
- 明确角色边界与核心业务
- 控制功能范围，避免做得过大

### 2. 最小数据库设计

文件：`02_最小数据库设计.md`

适合用途：

- 快速建库
- 明确最少需要哪些主表
- 统一核心字段和状态值

### 3. 本地演示脚本

文件：`03_本地演示脚本.md`

适合用途：

- 课堂演示
- 答辩准备
- 检查主流程是否闭环

### 4. 开发落地建议

文件：`04_开发落地建议.md`

适合用途：

- 确定技术路线
- 规划前后端项目结构
- 按阶段推进开发工作

### 5. 前后端接口清单与字段约定

文件：`05_前后端接口清单与字段约定.md`

适合用途：

- 给前后端协作者统一字段和状态口径
- 给 AI 生成代码时提供明确约束
- 让页面、接口和数据库之间更容易对齐

## 推荐阅读顺序

1. 先看 `01_项目概述与功能范围.md`
2. 再看 `02_最小数据库设计.md`
3. 然后看 `04_开发落地建议.md`
4. 再看 `05_前后端接口清单与字段约定.md`
5. 最后用 `03_本地演示脚本.md` 检查演示流程

## 精简原则

本次整理没有保留所有原始 AI 生成材料，而是按照以下原则收敛：

- 同类内容只保留一份主文档
- 优先保留能直接指导实现和演示的内容
- 删除泛化教程与重复建议
- 尽量让每份文档职责单一、边界清晰
