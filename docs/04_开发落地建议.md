# 开发落地建议

## 1. 推荐技术路线

如果课程没有强制要求 Java 后端，推荐采用：

- 前端：`Vue 3` + `Vite` + `Vue Router` + `Pinia` + `Axios`
- 后端：`FastAPI` + `SQLAlchemy`
- 数据库：`MySQL 8.0`

这样做的优点：

- 前后端分离清晰
- 开发速度快
- 本地演示稳定
- 适合课程实践规模

如果老师强制要求 Java，则可改为：

- 前端：`Vue 3`
- 后端：`Spring Boot`
- 数据库：`MySQL 8.0`

## 3. 开发顺序建议

### 第一阶段：初始化环境

- 创建前端项目
- 创建后端项目
- 创建数据库
- 建立基础目录结构

### 第二阶段：登录与角色主页

- 完成登录接口
- 完成登录页
- 完成三类主页

### 第三阶段：收件流程

- 快递员新增收件记录
- 普通用户查看自己的快递
- 普通用户输入快递单号和取件码完成自助取件

### 第四阶段：发件流程

- 普通用户提交发件申请
- 管理员审核并分配快递员
- 快递员确认揽件

### 第五阶段：异常处理

- 管理员新增异常记录
- 管理员处理异常并更新状态

### 第六阶段：整理与答辩准备

- 统一按钮和菜单名称
- 准备演示数据
- 按演示脚本完整跑通流程

## 4. 最少需要的后端模块

- 登录模块
- 用户模块
- 收件模块
- 发件模块
- 异常模块
- 记录查询模块

## 5. 最少需要的前端页面

- 登录页
- 普通用户主页
- 快递员主页
- 管理员主页
- 快递员收件入库页
- 普通用户取件页
- 发件申请页
- 发件管理页
- 快递员任务页
- 异常处理页
- 记录查看页

## 6. 初学者最重要的实现原则

- 先做能跑的，不先做好看的
- 先做主流程，不先做扩展功能
- 一次只做一个模块
- 每完成一块就马上测试
- 文档、数据库、前后端字段命名保持一致

## 7. 当前阶段不建议优先做的内容

- 扫码功能
- 消息通知
- 文件导入导出
- 复杂报表
- 复杂权限体系
- 小程序接入

## 8. 一句话总结

**对课程项目来说，最稳妥的做法是先把登录、收件、发件和异常处理这几条主线做通，再去考虑页面美化和扩展能力。**
