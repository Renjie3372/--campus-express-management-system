## 3. 我建议的技术路线

因为课程要求是前后端分离，所以项目需要至少拆成：

- 前端项目
- 后端项目
- 数据库

在你当前软件环境下，并结合你已经确认的选择，我建议直接采用下面这一套固定方案。

## 3.1 最终方案：Vue + Spring Boot + MyBatis-Plus + MySQL

这是当前文档的唯一推荐方案。

### 前端

- `Vue 3`
- `Vite`
- `JavaScript`
- `Vue Router`
- `Pinia`
- `Axios`
- `Element Plus`

### 后端

- `JDK 21`
- `Spring Boot`
- `MyBatis-Plus`
- `Maven`

### 数据库

- `MySQL 8.0`

### 原因

- 前后端职责清晰，适合课程实践中的分角色演示
- `Element Plus` 很适合后台管理系统页面，表格、表单、弹窗组件齐全
- `Spring Boot + MyBatis-Plus` 更符合常见 Java 课程项目口径，便于答辩展示
- `MyBatis-Plus` 在基础增删改查、分页和条件查询上更省事，适合新手快速落地
- `JDK 21` 已经在本机存在，后续环境统一更方便

## 3.2 当前推荐结论

本项目后续默认采用：

**前端：Vue 3 + Vite + JavaScript + Vue Router + Pinia + Axios + Element Plus**  
**后端：Spring Boot + MyBatis-Plus + Maven + JDK 21**  
**数据库：MySQL 8.0**

这套组合最适合“本地可运行、结构清晰、课程答辩稳定”的目标。

## 6. 基本系统架构设想

如果由我来实现，我建议系统结构如下。

## 6.1 总体架构

```text
前端浏览器
   ↓
Vue 前端项目
   ↓ HTTP/JSON
Spring Boot 后端
   ↓
MySQL 数据库
```

## 6.2 前端架构

前端建议拆为以下部分：

- 页面层
- 路由层
- 状态管理层
- 接口请求层
- 公共组件层

### 页面层建议

- 登录页
- 普通用户主页
- 快递员主页
- 管理员主页
- 收件管理页
- 出库管理页
- 发件申请页
- 发件管理页
- 异常处理页
- 历史记录页
- 账号信息页

### 路由层建议

作用：

- 管理页面跳转
- 做登录拦截
- 做角色访问限制

### 状态管理层建议

建议保存：

- 当前登录用户信息
- 当前用户角色
- token 或登录状态

### 接口请求层建议

作用：

- 封装登录接口
- 封装收件、发件、异常处理相关接口
- 统一错误提示

## 6.3 后端架构

后端建议采用分层结构：

```text
Controller / Router 层
Service 层
Repository / DAO 层
Database 层
```

### Controller / Router 层

负责：

- 接收前端请求
- 返回 JSON 数据

### Service 层

负责：

- 编写业务逻辑
- 组织收件、发件、异常流程

### Repository / DAO 层

负责：

- 与数据库交互
- 查询、插入、更新数据

### Database 层

负责：

- 建表
- 管理数据连接

## 6.4 后端模块建议

建议后端拆为以下模块：

- 用户模块
- 登录模块
- 收件模块
- 发件模块
- 异常模块
- 记录查询模块

### 用户模块

包含：

- 查询账号信息

### 登录模块

包含：

- 登录校验

### 收件模块

包含：

- 新增收件记录

- 完成出库

### 发件模块

包含：

### 异常模块

包含：

- 新增异常记录

### 记录查询模块

包含：
